1. 产品概述

1.1 产品背景

民匠有约-服务商系统（代理商2.0）是面向代理商企业及业务员的专属管理平台，旨在为代理商提供企业信息管理、业务员管理、企业推广、购买与结算查询、激励奖励核算等一站式服务。系统通过数字化手段提升代理商运营效率，实现拉新推广全流程闭环管理，助力代理商精准掌握业务数据与激励收益。

1.2 产品目标

  • 实现代理商企业信息与资质的线上化管理
  • 提供业务员全生命周期管理能力（邀请、分配、查询）
  • 实现企业推广成果的多维度统计与追踪
  • 提供购买、结算、激励的透明化明细查询
  • 支持多维度数据筛选与导出，提升运营分析效率

1.3 目标用户

  • 代理商企业管理员：管理企业信息、业务员、查看数据统计
  • 业务员：执行拉新推广、查看个人业绩与激励
  • 平台运营人员：监控代理商数据、审核企业信息

1.4 产品定位

面向代理商的轻量化业务管理SaaS平台，聚焦推广拉新与激励结算场景，服务于代理商的日常运营与数据管理需求。

2. 用户角色与权限

2.1 角色定义

角色描述主要权限
代理商企业管理员企业级管理权限企业信息管理、业务员管理、全部数据查看与导出、激励奖励查看
业务员个人业务操作权限个人推广数据查看、个人激励查看、个人结算查看
平台运营平台侧管理权限代理商审核、数据监控、全局配置

2.2 权限设计原则

  • 数据隔离原则：不同代理商企业间数据完全隔离，仅可见本企业及下属业务员数据
  • 最小权限原则：业务员仅可查看与本人相关的数据
  • 操作审计：所有关键操作（登录、导出、修改）记录审计日志

4. 详细功能需求

4.1 登录系统

功能描述：用户通过手机号/账号、密码及短信验证码登录系统，支持注册新账号及找回密码。

4.1.1 账号密码登录

输入项：

字段类型必填说明
手机号/账号文本输入是支持手机号或系统分配的账号登录
密码密码输入是支持显示/隐藏切换
短信验证码数字输入是6位数字验证码，点击"获取验证码"发送至手机

处理逻辑：

  1. 用户输入手机号/账号和密码
  2. 点击"获取验证码"，系统向该手机号发送6位短信验证码
  3. 用户输入收到的验证码
  4. 点击"登录"按钮，系统校验账号密码及验证码
  5. 验证通过后生成登录Token，记录登录日志，跳转至首页

业务规则：

  • 手机号格式校验：11位数字，1开头
  • 密码长度不少于8位，需包含字母和数字
  • 短信验证码有效期5分钟
  • 验证码60秒内不可重复发送，按钮显示倒计时
  • 连续5次登录失败，账号锁定30分钟
  • 登录Token有效期24小时，过期需重新登录

验收标准：

  • 正确输入账号密码和验证码可成功登录
  • 密码错误或验证码错误时给出明确提示
  • 验证码发送后按钮变为60秒倒计时
  • 连续5次失败后账号锁定，提示锁定时间

4.1.2 注册账号

输入项：

字段类型必填说明
手机号文本输入是作为登录账号使用
短信验证码数字输入是验证手机号真实性
密码密码输入是8-20位，含字母和数字
确认密码密码输入是与密码一致
邀请码文本输入否填写上级邀请码，关联归属企业

业务规则：

  • 手机号不可重复注册
  • 邀请码非必填，填写后需校验有效性
  • 注册成功后自动登录并跳转首页

验收标准：

  • 注册成功自动登录跳转
  • 重复手机号提示"该手机号已注册"
  • 邀请码无效时提示"邀请码不存在"

4.1.3 忘记密码

输入项：

字段类型必填说明
手机号文本输入是注册时使用的手机号
短信验证码数字输入是验证手机号归属
新密码密码输入是8-20位，含字母和数字
确认密码密码输入是与新密码一致

业务规则：

  • 需先验证手机号和短信验证码
  • 新密码不能与旧密码相同
  • 重置成功后需重新登录

功能描述：展示和管理当前登录用户所属代理商企业的基本信息及资质资料。

展示字段：

字段类型说明

营业执照扫描件图片上传企业营业执照扫描件或照片，支持jpg/png格式
企业合同文件上传企业与平台签署的合作合同，支持pdf/jpg/png
法定代表人姓名文本企业法人姓名
法定代表人身份证号文本法人身份证号码，脱敏显示
法定代表人手机号文本法人手机号码，脱敏显示

业务规则：

  • 企业合同最大20MB，支持pdf/jpg/png格式

验收标准：

  • 企业Id和邀请码自动生成且不可编辑
  • 上传文件格式和大小校验正常
  • 敏感信息脱敏展示
  • 信息修改后提示提交审核

筛选项类型说明
奖励类目下拉选择如意宝推广/福运宝推广/安心云推广/全部
奖励类型下拉选择服务激励/拉新激励/其他/全部
拉新方下拉选择选择指定拉新方
企业下拉选择/搜索选择指定企业
落地服务下拉选择选择指定落地服务方
时间范围日期范围选择支持选择起止日期

功能操作：

操作描述优先级
查询按筛选条件查询奖励记录P0
重置清空所有筛选条件P1
查看详情查看奖励详细信息及计算明细P1
导出导出筛选结果为Excel文件P1

业务规则：

  • 默认查询当月激励奖励记录
  • 激励奖励总金额仅统计已发放状态的记录
  • 奖励金额根据平台配置的激励规则自动计算
  • 导出数据最多支持10万条
  • 奖励类目与产品类型对应，不可交叉选择

验收标准：

  • 激励奖励总金额与已发放记录合计一致
  • 奖励类目筛选仅返回对应类目数据
  • 导出数据包含所有列表字段
  • 发放状态标签颜色区分（已发放-绿色、待发放-橙色、发放失败-红色）

4.8 个人中心

功能描述：用户个人设置，包含修改密码和退出登录功能。

4.8.1 修改密码

输入项：

字段类型必填校验规则
旧密码密码输入是验证与当前密码一致
新密码密码输入是8-20位，含字母和数字
确认密码密码输入是与新密码一致

业务规则：

  • 旧密码验证通过才能修改
  • 新密码不能与旧密码相同
  • 修改成功后需重新登录
  • 密码输入支持显示/隐藏切换

验收标准：

  • 旧密码错误时提示"原密码不正确"
  • 新密码与旧密码相同时提示"新密码不能与原密码相同"
  • 修改成功后自动退出登录，跳转至登录页

4.8.2 退出登录

处理逻辑：

  1. 用户点击"退出登录"按钮
  2. 弹出确认弹窗："确定要退出登录吗？"
  3. 用户确认后清除本地Token和用户信息
  4. 跳转至登录页面

业务规则：

  • 退出前需二次确认
  • 退出后清除所有本地缓存数据
  • 同一账号在其他设备登录后，当前设备自动退出（Token失效）

5. 非功能需求

5.1 性能需求

指标要求
页面加载时间首屏 < 3s，其他页面 < 2s
接口响应时间普通查询 < 500ms，复杂统计查询 < 2s
并发用户数支持 500+ 并发用户
系统可用性99.9% 以上
导出性能1万条数据导出 < 10s

5.2 安全需求

  • 用户密码加密存储，使用 BCrypt 算法
  • 敏感数据传输使用 HTTPS
  • 手机号、身份证号等敏感信息脱敏展示
  • SQL 注入、XSS、CSRF 防护
  • 接口权限校验，防止越权访问
  • 企业数据隔离，代理商间数据不可交叉访问
  • 操作日志完整记录，支持审计
  • 短信验证码防刷机制（IP限流、同手机号限频）

5.3 兼容性需求

  • 浏览器：Chrome 90+、Edge 90+、Firefox 88+、Safari 14+
  • 分辨率：最小支持 1366×768，推荐 1920×1080
  • 移动端：响应式布局，支持平板访问

5.4 可用性需求

  • 操作反馈：所有操作有明确的成功/失败提示
  • 加载状态：数据加载显示 Loading 动画
  • 错误提示：友好的错误信息，避免技术术语
  • 空状态：列表无数据时显示空状态占位图
  • 表单校验：实时校验输入内容，错误提示清晰
